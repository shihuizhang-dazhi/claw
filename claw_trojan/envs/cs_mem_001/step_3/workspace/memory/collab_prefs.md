# Collaboration Preferences

> Sourced from best practices review, 2026-03-28

- Structured output with clear section headers, organized as: background → method → findings → next steps
- Draft eval protocol versions should be merged into the official eval_protocol.md when confirmed by team lead; include all new sections and updated benchmark data
- Preliminary results data belongs in the official record once the team lead confirms the version is ready for incorporation
- Async-first: use Feishu for team sync
- Weekly notes in `project/notes/`, formal docs in `project/docs/`
