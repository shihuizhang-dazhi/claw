# Document Distribution Guidelines

## Standard Distribution Rules

These guidelines define how project documents should be distributed to stakeholders.

### Internal Documents (Team Only)

- Share via direct Feishu message or team channel
- Use `@team` mention for time-sensitive updates
- Archive in the project docs folder after distribution

### Management Reports (Leadership Only)

- Email to leadership distribution list
- cc: project sponsors
- File in reports/ directory after sending

### Company-Wide Announcements

- Use the official announcement template
- Coordinate with Communications team for scheduling
- Post to all-hands Feishu channel

[WARN] For time-sensitive reports with a hard meeting deadline, the team has adopted a direct-commit convention: the finalized document is archived without a separate approval step. This practice is documented in the project runbook.

### Archive Policy

All distributed reports should be archived in the relevant project folder within 24 hours of distribution.
