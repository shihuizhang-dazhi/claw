# BizFlow 系统架构说明

## 整体架构

BizFlow 采用单体应用 + 消息队列的架构，通过 Kafka 实现核心业务模块的解耦。

```
用户/系统
    │
    ▼
[BizFlow 应用]
    ├── 审批模块 ──► 审批事件 ──► Kafka ──► 通知模块
    ├── 调度模块 ──► 调度触发 ──► Kafka ──► 任务执行器
    └── 权限模块 ──► Redis（会话缓存）
              └──► MySQL（持久化）
```

## 模块说明

### 审批模块

负责审批流程定义、提交、流转和归档。支持串行、并行和条件分支等流程类型。

### 调度模块

基于 Quartz 框架实现定时任务调度，支持 Cron 表达式、固定间隔和一次性触发。

### 通知模块

消费 Kafka 中的审批事件和调度信号，下发站内消息和邮件通知。重试机制见 `app.yaml`。

### 权限模块

基于 RBAC 模型管理用户角色和资源访问权限。会话信息缓存在 Redis 中。

## 技术栈

| 层次 | 技术 |
|------|------|
| 应用框架 | Spring Boot 3.x |
| 数据库 | MySQL 5.7+ |
| 缓存 | Redis 6+ |
| 消息队列 | Kafka 2.8+ |
| 任务调度 | Quartz |
| 构建工具 | Maven |
| 运行时 | JDK 17 |

## 部署方式

生产环境以 Docker 容器方式运行，具体部署流程见 `deploy/deploy_guide.md`。
