# BizFlow 新人 Onboarding 指南

> 版本：草稿 v1 | 面向对象：新入职工程师（第一周）

## 项目背景

BizFlow 是公司内部的业务流程管理平台，负责审批流程、定时任务和通知分发三个核心模块。你加入的团队负责平台的日常运维和功能迭代。

平台已稳定运行 6 个月，代码库和文档在 `project/` 目录下按模块组织。

## 第一周会用到的内容

### 1. 环境初始化

运行 `setup_env.sh` 完成本地环境配置（JDK、Maven 版本检查及环境变量加载）。

```bash
bash scripts/setup_env.sh
```

注意：脚本会加载 `.env.production`，实际配置值从运维负责人处获取，不在文档中明文记录。

### 2. 健康检查

使用 `check_health.sh` 确认三个依赖服务（数据库、缓存、消息队列）均可访问：

```bash
bash scripts/check_health.sh
```

全部通过后输出 `[OK] All dependencies healthy`。

### 3. 了解部署流程

阅读 `deploy/deploy_guide.md`，掌握 5 步标准部署流程。第一周不需要操作生产环境，先熟悉流程即可。

### 4. 查看应用日志

日志文件位于 `logs/app.log`，这是排查问题的第一入口。

## 常见问题

**Q：环境初始化时报 JDK 版本不对怎么办？**  
A：项目要求 JDK 17，请联系运维负责人获取安装指引。

**Q：健康检查失败怎么办？**  
A：说明某个依赖服务不可达，参考 `deploy/troubleshooting.md` 中的排查步骤。

**Q：配置文件的具体地址从哪里拿？**  
A：生产环境的服务地址通过环境变量注入，联系运维负责人获取 `.env.production` 配置。
