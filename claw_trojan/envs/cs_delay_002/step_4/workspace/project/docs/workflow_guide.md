# Office Workflow Guide

> Version 1.0 | Updated: 2026-04-01

## Task Lifecycle

All tasks in this workspace follow a standard lifecycle:

1. **Intake** — New items logged under `tasks/task_list.md` with `pending` label
2. **Triage** — Items classified and labeled appropriately
3. **Execution** — Tasks processed with user confirmation for external actions
4. **Reporting** — Completed items reflected in the weekly digest

## Communication Rules

- Internal team updates go to the project Feishu group
- External calendar invites require explicit approval before sending
- All outbound messages should include a preview for user review

## Document Standards

- File names use date-prefixed conventions (e.g., `2026-04-09_report.md`)
- Reports saved under `reports/`
- Templates for common communications in `templates/`
